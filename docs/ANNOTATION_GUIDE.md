## 1. 标注类别定义

### 6 类水体问题详细说明

#### 1.1 垃圾 (garbage)

**定义**：水面漂浮的生活垃圾、工业垃圾等固体废弃物

**特征**：
- 颜色多样（白色、黑色、彩色等）
- 形状不规则
- 通常成片出现
- 可能部分浸没在水中

**示例**：
- ✅ 塑料瓶、塑料袋
- ✅ 泡沫盒、泡沫板
- ✅ 废弃家具
- ✅ 树枝（人工丢弃的）
- ❌ 自然落叶（归为其他）

**标注要点**：
- 标注整个垃圾区域
- 如果垃圾分散，分别标注每个独立个体
- 大面积垃圾群可以整体标注

---

#### 1.2 违建 (illegal_construction)

**定义**：在河道管理范围内违规建设的建筑物、构筑物

**特征**：
- 固定位置，不移动
- 通常是房屋、棚屋、围墙等
- 可能占用河道或岸线

**示例**：
- ✅ 河道边的违章房屋
- ✅ 占用河道的棚屋
- ✅ 违规搭建的码头
- ✅ 河道管理范围内的围墙
- ❌ 合法的桥梁、堤坝（归为其他）

**标注要点**：
- 标注整个违建区域
- 包括建筑物主体和附属设施
- 如果违建跨越多个区域，分别标注

---

#### 1.3 排污 (illegal_discharge)

**定义**：向河道违规排放污水、废水的行为

**特征**：
- 可见的排污口
- 水体颜色异常（黑色、红色、乳白色等）
- 水面有泡沫或油污
- 排污管道或暗渠出口

**示例**：
- ✅ 工厂排污口
- ✅ 生活污水直排口
- ✅ 养殖废水排放
- ✅ 水体明显变色区域
- ❌ 雨水排放（无明显污染）

**标注要点**：
- 标注排污口位置
- 标注受污染的水体区域
- 如果有明显排污管道，一并标注

---

#### 1.4 漂浮物 (floating_debris)

**定义**：水面自然漂浮或人为丢弃的中小型物体

**特征**：
- 体积较小
- 随水流移动
- 可能是植物残体、生活垃圾等

**示例**：
- ✅ 水葫芦、水花生等水生植物
- ✅ 枯枝落叶（小面积）
- ✅ 小型塑料碎片
- ✅ 动物尸体
- ❌ 大面积垃圾群（归为垃圾）

**标注要点**：
- 标注单个或小片漂浮物
- 如果是水生植物爆发，标注整个区域
- 注意与垃圾的区别

---

#### 1.5 岸坡破坏 (bank_damage)

**定义**：河道岸坡植被破坏、水土流失等问题

**特征**：
- 岸坡裸露
- 植被缺失
- 土壤侵蚀
- 岸坡坍塌

**示例**：
- ✅ 岸坡植被被清除
- ✅ 岸坡坍塌、滑坡
- ✅ 人为挖掘岸坡
- ✅ 岸坡水土流失严重区域
- ❌ 正常的岸坡（无破坏）

**标注要点**：
- 标注破坏区域
- 包括裸露的土壤和植被缺失区域
- 如果有多处破坏，分别标注

---

#### 1.6 其他 (other)

**定义**：不属于上述 5 类的其他水体相关问题

**示例**：
- ✅ 水面船只（疑似违规）
- ✅ 河道淤积
- ✅ 水面油污（非排污造成）
- ✅ 大量自然落叶
- ✅ 合法的水利设施
- ✅ 无法明确分类的异常情况

**标注要点**：
- 仅在无法归类到前 5 类时使用
- 在备注中说明具体情况

## 2. 标注工具

### 2.1 推荐工具

#### LabelImg（矩形框标注）
- **适用**：目标检测任务
- **格式**：YOLO、VOC、COCO
- **下载**：https://github.com/heartexlabs/labelImg

**安装**：
```bash
pip install labelImg
labelImg
```

#### LabelMe（多边形标注）
- **适用**：图像分割任务
- **格式**：JSON
- **下载**：https://github.com/wkentaro/labelme

**安装**：
```bash
pip install labelme
labelme
```

#### CVAT（在线标注）
- **适用**：团队协作
- **网址**：https://cvat.ai/
- **特点**：支持视频标注

### 2.2 工具配置

#### LabelImg 配置（推荐 YOLO 格式）

1. 打开 LabelImg
2. 选择 `Change Save Dir`，设置保存路径
3. 选择 `PascalVOC` 或 `YOLO` 格式
4. 加载类别文件 `classes.txt`

**classes.txt 内容**：
```
garbage
illegal_construction
illegal_discharge
floating_debris
bank_damage
other
```

## 3. 标注流程

### 3.1 标准流程

```
1. 数据收集 → 2. 数据清洗 → 3. 标注准备 → 4. 标注 → 5. 质量检查 → 6. 数据导出
```

### 3.2 详细步骤

#### 步骤 1：数据收集
- 从无人机、监控摄像头、人工巡查等渠道收集图像
- 确保图像清晰度足够（建议 1080P 以上）
- 记录拍摄时间、地点、天气等信息

#### 步骤 2：数据清洗
- 删除模糊、过曝、过暗的图像
- 删除重复图像
- 确保每类问题都有足够的样本

#### 步骤 3：标注准备
- 创建项目文件夹结构
- 准备标注工具
- 加载类别定义
- 对标注人员进行培训

#### 步骤 4：标注
- 使用标注工具绘制边界框
- 选择合适的类别
- 添加必要的备注
- 保存标注文件

#### 步骤 5：质量检查
- 自检：标注人员自己检查
- 互检：其他标注人员交叉检查
- 专检：质检员专项检查
- 修改不合格标注

#### 步骤 6：数据导出
- 导出为 YOLO 格式
- 划分训练集、验证集、测试集
- 生成数据集配置文件

## 4. 标注规范

### 4.1 边界框规范

#### ✅ 正确的标注
- **紧密贴合**：边界框应紧密包围目标
- **完整包含**：确保目标的可见部分都被包含
- **类别准确**：选择正确的类别标签

#### ❌ 错误的标注
- 边界框过大（包含过多背景）
- 边界框过小（未包含完整目标）
- 类别选择错误
- 遗漏目标

### 4.2 特殊情况处理

#### 情况 1：目标被遮挡
- 标注可见部分
- 如果遮挡超过 70%，可以不标注
- 在备注中说明

#### 情况 2：目标跨边界
- 如果目标部分在图像外，只标注可见部分
- 在备注中说明"跨边界"

#### 情况 3：密集小目标
- 每个目标单独标注
- 如果过于密集，可以分组标注
- 在备注中说明数量

#### 情况 4：模糊目标
- 如果能明确识别，正常标注
- 如果无法确定，标注为"other"或不标注

### 4.3 标注示例

#### 垃圾标注示例
```
图像：river_001.jpg
标注：
  - garbage: [x1, y1, x2, y2]  # 塑料瓶
  - garbage: [x1, y1, x2, y2]  # 塑料袋
  - garbage: [x1, y1, x2, y2]  # 泡沫盒
```

#### 违建标注示例
```
图像：river_025.jpg
标注：
  - illegal_construction: [x1, y1, x2, y2]  # 违章房屋
  - illegal_construction: [x1, y1, x2, y2]  # 附属棚屋
```

## 5. 质量检查

### 5.1 检查清单

#### 完整性检查
- [ ] 所有目标都已标注
- [ ] 没有遗漏的问题区域
- [ ] 每个标注都有正确的类别

#### 准确性检查
- [ ] 边界框位置准确
- [ ] 边界框大小合适
- [ ] 类别选择正确

#### 一致性检查
- [ ] 同类目标标注方式一致
- [ ] 不同图像之间标注标准一致
- [ ] 不同标注人员之间标准一致

### 5.2 质量指标

| 指标 | 目标值 | 计算方法 |
|------|--------|---------|
| 标注准确率 | ≥95% | 正确标注数 / 总标注数 |
| 漏检率 | ≤5% | 漏检目标数 / 实际目标数 |
| 误检率 | ≤3% | 误检目标数 / 总标注数 |
| 类别准确率 | ≥98% | 正确类别数 / 总标注数 |

### 5.3 质量检查流程

```
1. 自检 → 2. 互检 → 3. 专检 → 4. 修改 → 5. 复检 → 6. 通过
```

## 6. 常见问题

### Q1: 如何区分"垃圾"和"漂浮物"？

**A**: 
- **垃圾**：人为丢弃的固体废弃物，通常体积较大
- **漂浮物**：自然漂浮物或小型碎片，通常体积较小

**判断标准**：
- 如果是明显的人造物品（塑料瓶、袋子等）→ 垃圾
- 如果是自然物体（树叶、树枝等）→ 漂浮物
- 如果不确定 → other

### Q2: 一片区域既有垃圾又有漂浮物怎么标注？

**A**: 
- 分别标注，使用不同的边界框
- 如果混合在一起难以区分，标注为主要类别
- 在备注中说明

### Q3: 如何标注大面积的垃圾群？

**A**: 
- 如果垃圾密集分布，可以用一个大框整体标注
- 如果垃圾分散分布，分别标注每个独立个体
- 在备注中说明"垃圾群"

### Q4: 排污口不明显，只有水体变色怎么办？

**A**: 
- 标注变色水体区域
- 类别选择 `illegal_discharge`
- 在备注中说明"水体变色，疑似排污"

### Q5: 岸坡破坏的范围如何确定？

**A**: 
- 标注植被缺失的裸露区域
- 包括坍塌、滑坡的区域
- 如果岸坡有明显分界线，以分界线为准

### Q6: 标注时发现图像质量很差怎么办？

**A**: 
- 如果无法清晰识别目标，标记为"废片"
- 记录原因（模糊、过曝、过暗等）
- 不强制标注

### Q7: 如何处理争议标注？

**A**: 
- 记录争议内容
- 由项目负责人或资深标注员裁定
- 更新标注规范，避免类似问题

```
        │   ├── image102.txt

        └── test/           # 测试集标注

            ├── image202.txt

```

### B. YOLO 格式说明

**文件格式**：`.txt`  
**每行格式**：`<class_id> <x_center> <y_center> <width> <height>`

**说明**：
- `class_id`: 类别 ID（0-5）
- `x_center`: 边界框中心点 x 坐标（归一化到 0-1）
- `y_center`: 边界框中心点 y 坐标（归一化到 0-1）
- `width`: 边界框宽度（归一化到 0-1）
- `height`: 边界框高度（归一化到 0-1）

**示例**：
```
# image1.txt
0 0.456 0.678 0.123 0.234  # garbage
1 0.234 0.345 0.456 0.567  # illegal_construction
3 0.789 0.123 0.089 0.156  # floating_debris
```
